# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this project is

CFnew (终端 v2.9.6) is a single-file Cloudflare Worker that exposes VLESS / Trojan / xhttp proxy protocols over WebSocket, plus a graphical admin UI and a subscription generator for ten client formats (Clash, Surge, Sing-Box, Loon, QuantumultX, V2Ray, Shadowrocket, Stash, NekoRay, V2RayNG). It is forked from `zizifn/edgetunnel`.

The worker has no `package.json`, no bundler, no local test suite. Everything ships as one large file deployed straight to Cloudflare Workers / Pages.

## Source vs. obfuscated output (important)

The repository uses two files with Chinese names for the same worker:

| File | Role | Edit it? |
| :--- | :--- | :--- |
| `明文源吗` | Plaintext source — the file humans edit | **Yes — this is the source of truth** |
| `少年你相信光吗` | Obfuscated build artifact (~1.4 MB, single line) | **No — generated by CI** |
| `snippets` | Separate, smaller variant for Cloudflare Snippets deployment | Edit only if working on the snippets variant |

`.github/workflows/obfuscate.yml` runs on every push that touches `明文源吗`, runs `javascript-obfuscator` over it (string-array + base64 + mangled identifiers, see the workflow for the exact options), and commits the result back as `少年你相信光吗`. **Never hand-edit `少年你相信光吗`** — the next push to the source will overwrite it.

`.github/workflows/test.yml` runs on tag push: it copies `少年你相信光吗` → `_worker.js`, zips it with `wrangler.toml`, and publishes a GitHub Release named `Pages.zip` for users to upload to Cloudflare Pages.

## Deployment model

There is no `wrangler deploy` flow checked in. `wrangler.toml` only pins `compatibility_date = "2026-01-20"`, which **must** be set on the Cloudflare side after upload (the README opens with this warning — Pages and Workers both need the date selected manually in the dashboard, otherwise the worker fails at runtime).

Two deployment paths users follow:
1. **Pages**: upload the `Pages.zip` release (contains `_worker.js` + `wrangler.toml`).
2. **Workers**: paste `少年你相信光吗` (or `明文源吗` in plaintext) into the Workers editor.

A KV namespace bound as `C` is optional but enables the graphical config (`/{UUID}` page persists settings to `kvStore.get('c')` / `put('c', ...)`). Without `C`, configuration falls back to env vars only.

## Architecture of `明文源吗`

Single-file ES module Worker. Top-level shape:

- **Lines 1–104** — module-level mutable config (`at` UUID, `ev`/`et`/`ex` protocol toggles, `enableECH`, `customDNS`, `customECHDomain`, `scu` subconverter URL, `backupIPs`, `directDomains`, base64-encoded error strings `E_*`).
- **Lines 105–310** — KV helpers (`initKVStore`, `loadKVConfig`, `saveKVConfig`, `getConfigValue`, `setConfigValue`), region detection (`detectWorkerRegion`, `getBestBackupIP`, `getSmartRegionSelection`), and `parseAddressAndPort`.
- **Line 312 — `export default { async fetch(request, env, ctx) { … } }`** is the only entry point. It dispatches by `url.pathname`:
  - `/{user}/api/config` — KV config CRUD (`明文源吗:490`).
  - `/{user}/api/preferred-ips` — preferred-IP management API (`明文源吗:524`).
  - `/{user}/region`, `/{user}/test-api` — admin probes (`明文源吗:581`, `明文源吗:641`).
  - `/{user}` and `/{user}/sub` — admin UI / subscription generator (handed to `handleSubscriptionPage` and `handleSubscriptionRequest`).
  - WebSocket upgrades — handed to `handleWsRequest` (`明文源吗:1986`), which parses the VLESS/Trojan/xhttp header (`parseWsPacketHeader` `明文源吗:2152`, `parseTrojanHeader` `明文源吗:4955`), then `forwardTCP` / `forwardUDP` over a `connect()` socket from `cloudflare:sockets`. SOCKS5 fallback is in `establishSocksConnection` (`明文源吗:2222`).
- **Lines ~970–4900** — embedded HTML/CSS/JS for the admin terminal and latency-test UI is returned as inline templates from `handleSubscriptionPage`. When editing the UI, you are editing string literals inside this file.
- **Lines ~5000–6378** — xhttp protocol bits (`parseTrojanHeader`, `sha224Hash`, `validate_uuid_xhttp`, `parse_uuid_xhttp`, `concat_typed_arrays`, `xhttp_sleep`, `rightRotate`).

The "user" segment of the URL is matched against either the UUID (`u` env var, default `at` constant) or the custom path (`d` env var, persisted as `cp` in KV). Both are handled — never assume only one.

## Configuration precedence

`KV (binding C, key 'c') > env vars > module-level defaults`. When changing default behavior, pick the right layer:
- Module-level constants in `明文源吗` are the fallback when neither KV nor env are set. Changing one is a code change that requires re-obfuscation + redeploy.
- Env vars (`u`, `p`, `s`, `d`, `wk`, `ev`, `et`, `ex`, `tp`, `ech`, `yx`, `yxURL`, `scu`, `epd`, `epi`, `egi`, `qj`, `dkby`, `yxby`, `rm`, `ae`) are read in the fetch handler — see README for the full list and meanings.
- KV values overwrite env vars at runtime via `getConfigValue`. The admin UI writes here.

A fourth layer exists for **per-node** overrides (v2.9.4+): VLESS/Trojan share-link `path` query params (`p`, `wk`, `rm`, `s`) override KV/env for that single connection. These are parsed in `handleWsRequest` and threaded through `forwardTCP` as `reqFallback` / `reqRegion` / `reqRm` / `reqSocksConfig`. Subscription-level toggles like `ev`/`et`/`yx` cannot be overridden this way — they are evaluated before the WebSocket handshake.

## Region matching

Region selection (`getSmartRegionSelection` `明文源吗:268`) prefers same-region ProxyIP, then `getNearbyRegions` neighbors (e.g., DE → NL/SE/FI/GB), then any other. Setting `p` (custom ProxyIP) **fully bypasses** this logic — the README calls out that `p` and `wk` are mutually exclusive; if both are set, `p` wins.

## When making code changes

- Edit `明文源吗` only. Push, then let the `obfuscate.yml` workflow regenerate `少年你相信光吗`.
- The source's leading indentation is unusual (4 spaces on top-level statements because of how it was extracted). Match the surrounding style rather than reformatting.
- Many user-visible strings are duplicated in Chinese and Persian (Farsi) — `فارسی.md` is the Persian README. UI text branches on a `lang` cookie; if you add a string, add both translations or it will appear untranslated for one audience.
- Error strings are stored as `atob('...')` of base64 to keep them out of plaintext greps. Reuse the existing `E_*` constants instead of inlining new ones.
- The compatibility date `2026-01-20` is required for current `cloudflare:sockets` behavior; do not lower it.
